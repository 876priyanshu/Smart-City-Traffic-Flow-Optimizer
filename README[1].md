# Smart City Traffic Flow Optimizer

A Windows CMD-friendly project that combines a C++ routing engine with an interactive Leaflet.js map. It models a city road network, finds shortest routes with Dijkstra, simulates traffic (dynamic weights), and visualizes results on a clean web map.

## Features
- C++ core: Graph (adjacency list), Dijkstra, traffic simulator, JSON export
- Web UI: Leaflet.js map, modern CSS, responsive layout
- Interactive: Pick source/destination, view path, see travel time, history
- CMD-friendly: Build with MinGW, no external libs

## Build (Windows CMD / VS Code)
```cmd
cd SmartCityTrafficFlowOptimizer
mingw32-make
# OR
if not exist bin mkdir bin
g++ -std=gnu++14 -O2 -I src src\*.cpp -o bin\traffic.exe
```

## Run
- C++ console app:
  ```cmd
  bin\traffic.exe
  ```
- Web UI:
  - Open `web/index.html` in a browser.
  - The UI reads `data/graph.json` and `data/route.json` generated by the C++ app.

## Data files
- `data/city_map.txt` (small graph)
- `data/city_map_large.txt` (large graph)
- `data/graph.json` (exported for map)
- `data/route.json` (exported for route)
- `data/routes_history.txt` (append at runtime)
- `data/traffic_logs.txt` (simulator logs)

## VS Code
- .vscode/tasks.json: build task
- .vscode/launch.json: run configuration (external console)

## Architecture
- C++ loads graph, runs Dijkstra, simulates traffic, writes JSON.
- Web UI reads JSON, draws nodes/edges on Leaflet map, highlights route.
- No WebAssembly; simple JSON bridge for simplicity.

## Demo flow
1) Build and run C++ app once to export JSON.
2) Open web/index.html.
3) Select source/destination, click Find Route.
4) See path and time on map.
5) Run C++ simulator to update weights; refresh web UI to see changes.

## Notes
- Map uses OpenStreetMap tiles; internet required.
- Colors and animations work in any modern browser.
- CMD output uses ANSI colors; Windows Terminal recommended.
